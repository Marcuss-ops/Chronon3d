# CapCut Export Guide — Blessed Reference PNGs

**Purpose**: this guide tells a designer/maintainer how to export each case in
[`manifest.json`](manifest.json) from **CapCut Desktop** so the resulting PNG
can be placed in `tests/reference/capcut/<area>/reference/` and blessed as a
gold-standard reference.

**Policy reminder**:
- `reference/` is **blessed-only** (never auto-generated by Chronon3D).
- `current/` is auto-generated by Chronon3D and is gitignored.
- Never overwrite `reference/` with Chronon3D output.

---

## Prerequisites

- CapCut Desktop (the exact version should be recorded in the commit message).   - The same font files used by the manifest (they are bundled under `assets/fonts/`):
  - `assets/fonts/Inter-Bold.ttf`
  - `assets/fonts/Inter-Regular.ttf`
  - `assets/fonts/Poppins-Bold.ttf`
- The fonts installed in your OS (or loaded into CapCut) so the exact family
  and weight are available.
- A tool to verify PNG resolution and alpha channel (e.g. `file`, `identify` from ImageMagick, or any image viewer).

---

## Generic CapCut export steps

For every case below, follow this workflow:

1. **Create a new project** in CapCut Desktop with the exact resolution listed.
2. **Remove the default background/video track** so the canvas is transparent.
   - In the timeline, delete the default video/background layer.
   - The exported PNG must have a transparent background (alpha channel).
3. **Add a Text layer**.
4. **Set the text**, font, font size, tracking, and line spacing exactly as listed.
4. **Install/load the exact font** in CapCut before creating the text layer.
5. **Set alignment and vertical alignment** as listed.
6. **Position the text** using the placement/offset described. For `CanvasCenter'
   cases, center the text in the canvas. For `Absolute` cases, set the text anchor to the point described by
   `align` + `vertical_align` and place that anchor at the provided `(x, y)`
   coordinates from the top-left origin. For example, a subtitle case with
   `align: Center` and `vertical_align: Bottom` means: anchor point is the
   bottom-center of the text box, positioned at `(x, y)`.
7. **Disable any default animation/effect** unless the case explicitly lists one.
8. **Go to the requested frame** (for `effects` cases).
9. **Export the frame** as PNG with no compression/lossless settings:
   - Format: PNG
   - Background: transparent
   - Quality: lossless / 100 %
   - No watermark, no overlays, no timeline UI elements.
10. **Verify the exported PNG**:
    - Resolution matches the manifest.
    - Has an alpha channel (RGBA, not RGB).
    - The text is the only opaque content.
11. **Copy the PNG** to `tests/reference/capcut/<area>/reference/<id>.png`.
12. **Open a PR** following the README.md blessing workflow.

## Verification command

After copying the PNG into `reference/`, verify it quickly with:

```bash
# Check resolution and alpha channel
file tests/reference/capcut/static/reference/static_inter_bold_capcut.png
# Expected: PNG image data, 1920 x 1080, 8-bit/color RGBA, non-interlaced

# Compute PNG checksum for the commit message
sha256sum tests/reference/capcut/static/reference/static_inter_bold_capcut.png
```

Or with ImageMagick:

```bash
identify -verbose tests/reference/capcut/static/reference/static_inter_bold_capcut.png | \
  grep -E "Print size|Resolution|Colorspace|Alpha|Compression"

# Verify the background is actually transparent outside the text region
convert tests/reference/capcut/static/reference/static_inter_bold_capcut.png \
  -alpha extract -format "min=%[min] max=%[max] mean=%[mean]\n" info:
# Expected: max=255 where text is opaque, mean very low (mostly transparent)
```

## Commit message template

When blessing a new reference, use a commit message like:

```
seed(capcut): bless reference static_inter_bold_capcut from CapCut Desktop vX.Y.Z

- Tool: CapCut Desktop vX.Y.Z (pin the exact build/version used)
- Font: assets/fonts/Inter-Bold.ttf (Inter Bold, 700)
- Text: "CapCut"
- Size: 96 pt
- Tracking: 0
- Line spacing: 1.2
- Resolution: 1920x1080
- Placement: CanvasCenter
- Alignment: Center / Middle
- Background: transparent
- Encoding: PNG lossless RGBA
- PNG sha256: <sha256sum>
```

**Version pinning**: record the exact CapCut Desktop build in the commit
message. If a different version is used later, the reference may need to be
re-blessed because CapCut's renderer or defaults can change between versions.
