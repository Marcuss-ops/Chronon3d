# tests/baselines/renderer_before.txt

# §13.0 — Committed baseline of chronon3d_renderer_tests sources.
#
# This file is the single source of truth for the §13 series correctness
# check.  The §13 series splits the monolithic chronon3d_renderer_tests
# executable (defined in tests/renderer_tests.cmake) into 6 per-area
# targets:
#
#   chronon3d_effects_tests
#   chronon3d_camera_tests
#   chronon3d_golden_tests
#   chronon3d_precomp_tests
#   chronon3d_render_graph_tests
#   chronon3d_renderer_core_tests
#
# The §13.13 correctness gate sorts the lines of THIS file
# (before.sorted) and diffs it against the sorted union of the 6 new
# targets' source lists (after.sorted).  The diff MUST be empty: every
# .cpp file in the monolithic must appear in exactly one of the 6 new
# targets, and no .cpp file may be added or lost in the split.
#
# Format
# ------
# One path per line, relative to the repo root.  Alphabetically sorted
# (so the raw file == before.sorted modulo the comment header).  Comments
# (#) at the top describe the file; blank lines + non-# lines below the
# header are the source list and are diff-checked.
#
# Sources
# -------
# 85 .cpp files in total = 83 unconditional + 1 (${TEST_MAIN} expansion
# = tests/test_main.cpp, set in tests/CMakeLists.txt) + 1
# (${RENDERER_BLEND2D_TESTS} = tests/renderer/test_blend_pixel_nan.cpp,
# gated by CHRONON3D_USE_BLEND2D).
#
# The baseline is the union across all feature flag combinations.  Per-
# area targets at §13.7-13.12 may re-apply the same feature gates
# (CHRONON3D_USE_BLEND2D, etc.) — the diff still holds because the
# gated-out sources don't appear in either set at validation time.
# ============================================================================
