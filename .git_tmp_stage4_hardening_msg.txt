feat(builders): harden Sub-case 29 wiring probe with deterministic pre-build inspector

Cluster A Stage 4 hardening. Stage 4 (commit 37f1dada) shipped the
AnimatorResolver wiring for cinematic_text_camera. Sub-case 29 of
tests/test_text_preset_registry.cpp had a best-effort post-build probe
that depended on font-engine materialization returning a non-null
text_run_shape_handle().value pointer. If materialization returned
null (no real font on disk in CI), the check silently skipped and the
wiring evidence vanished.

This commit replaces the lenient probe with a deterministic pre-build
inspector: a new test-only accessor on LayerBuilder exposes the pending
TextRunSpec slots BEFORE build() consumes them, so the resolver's push
is observable on every test run regardless of font materialization
state.

Implementation:

- include/chronon3d/scene/builders/layer_builder.hpp
  - New inline accessor `pending_text_runs()` returning
    `std::vector<const PendingTextRun*>` by value.
  - Const-correctness: pointers are PendingTextRun const* so the
    TYPE SYSTEM prevents production-code mutation (not only docs).
  - Docstring covers: empty-vector case (Sub-cases 7-9 with lb.text
    path), one-allocation cost, lifetime caveats (push_back realloc
    invalidates pointers; destruction / move / reset also invalidates).
  - Header-only change: no .cpp touch keeps scope tight.

- tests/test_text_preset_registry.cpp
  - Sub-case 29 (Tier E) rewritten to use the pre-build inspector:
      1. Pre-build REQUIRE: lb.pending_text_runs() is empty BEFORE
         preset.builder() is called (proves inspector starts empty).
      2. After preset.builder() invocation, REQUIRE(pre.size() == 1)
         and REQUIRE(pre[0]->name == "camera_text").
      3. REQUIRE(pre[0]->params.animators.size() >= 1).
      4. Strict index-based CHECK:
         animators[0].id.rfind("ctc_rich_cinematic_text_camera", 0) == 0
         (proves the resolver pushed the wired entry EXACTLY at
         index 0; a future Stage 5+ resolver that inserts a
         different animator BEFORE the wired one fails loudly).
  - Tier E header comment updated to reflect the new deterministic
    pre-build contract.
  - Sub-cases 1-28 untouched: pre-existing assertions on
    built.nodes.size() / depth_offset / registry metadata still
    valid (lb.text(...) path does NOT touch m_text_runs).

Invariants preserved:

- Anti-circular-dep: 0 `#include ... content/` lines added.
- Scope discipline: 2 files touched (1 header + 1 test).
- Sub-cases 7-9 still pass: make_test_text_spec has no richness
  triggers so cinematic_text_camera falls through plain lb.text(...)
  path which creates zero m_text_runs entries.
- 22 entries preserved (Stage 3 count unchanged).
- Origin/main was ff-merged (PR #30 cmake-sdk-baseline, commit
  b5a4f6e8) into local main BEFORE this commit landed; no conflicts.
