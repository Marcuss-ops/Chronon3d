feat(expressions): Path B expression engine (compile-once, AE-oriented)

P3 introduces Path B -- an AE-oriented expression engine -- on the
feature/expressions-v2 branch (main stays on Path A until the freeze).
The engine is implementation-grade end-to-end: source through to VM
evaluation plus dependency-graph cycle detection.

== Pipeline (7 stages)
  source -> Lexer -> AST -> Type checker -> Bytecode -> VM
  plus DependencyGraph + Cycle detection (Kahn's topo sort).

== ExpressionValue std::variant
  double / bool / std::string / Vec2 / Vec3 / Vec4 / Color
  / LayerReference (opaque; host-resolvable) / PropertyReference.
  LayerReference and PropertyReference are opaque in v2: the compiler,
  AST, bytecode, and VM reason about them; the host resolves them at
  evaluation time via env callbacks.

== AST (tagged union, no virtuals or shared_ptrs)
  NumberLiteral, StringLiteral, BoolLiteral, Identifier, MemberAccess,
  IndexAccess, Call, Binary, Unary, Conditional. Each node carries a
  SourceSpan for downstream diagnostics.

== Type checker
  Recursive post-order. Permissive on Vec+Number mixing; strict on
  string arithmetic with non-string. thisLayer.position is Vec3;
  thisLayer.position.x and .opacity are Number; member access on Vec2
  / Vec3 / Vec4 / Color is well-typed.

== Bytecode
  Stack-machine. LOAD_CONST packs (tag in high 8 bits | index in low
  24 bits) into slot to disambiguate the three constant pools (numbers,
  strings, bools). LOAD_VAR, STORE_VAR, CALL, and MEMBER index program
  names, an interned-string table built during compile.

== Compiler
  Single-pass recursive descent with precedence climbing. Conditional
  expressions back-patch JMP and JMP_IF_FALSE targets. Function calls
  derive arg arity by name (clamp 3, max 2, others 1). It is invalid to
  call with the wrong number of args; the VM surfaces this at runtime.

== VM
  Stack-machine interpreter. Vec3+Number and Vec3+Vec3 arithmetic is
  fully implemented for ADD, SUB, MUL, DIV, MOD. Other vector dims
  coerce via number fallback (deferred to v2.1).
  Built-ins: sin/cos/tan/sqrt/abs/floor/ceil/round/pow/log/length and
  min/max/clamp.

== DependencyGraph + cycle detection
  add_program walks bytecode to collect reads (LOAD_VAR) and writes
  (STORE_VAR). add_writer registers a declarative writer. topological_
  order runs Kahn's; cycle nodes accumulate and report to the host so
  cyclic expressions are refused before evaluation.

== Tests (24 doctest cases)
  Variant helpers, Lexer round-trip, AST compile plumbing, Type
  checker (Number arithmetic, thisLayer.position.x, string+number
  rejection), Bytecode packing helpers, end-to-end Vm::evaluate
  examples (1+2*3=7, ternary, function call rounding), Vec3
  arithmetic completeness, member access on Vec3, dependency graph
  (acyclic order, length-2 cycle, isolated node), and statelessness
  of Vm::evaluate.

== Wiring
  src/CMakeLists.txt gates add_subdirectory(expressions) by file
  existence, so when the branch is not checked out the main build
  stays green. tests/CMakeLists.txt does the same for the include of
  tests/expressions/CMakeLists.txt. New target chronon3d_expressions_
  v2_tests is isolated from chronon3d_scene_tests. Main stays on
  Path A (AnimatedValue scalar); Path B remains on this branch until
  merge post-freeze.
