# Documentazione e ADR

- [ ] Definire una fonte unica per lo stato corrente.
- [ ] Sincronizzare `STATUS`, `ROADMAP` e `NEXT_STEPS`.
- [ ] Spostare la cronologia lunga fuori dai sorgenti.
- [ ] Creare ADR per render graph, runtime ownership, SDK e scope.
- [ ] Lasciare nei sorgenti solo invarianti ancora valide.
- [ ] Aggiornare documentazione e ticket nello stesso cambiamento.
- [ ] Distinguere implementato, compilato, testato e validato in CI.

## Completato quando

I documenti non si contraddicono e il codice non è appesantito da cronologia ormai superata.
